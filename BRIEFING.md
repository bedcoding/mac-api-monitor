# API Monitor — 프로젝트 브리핑

> 이 문서는 다음 클로드 세션의 시작점입니다.

---

## 1. 도구의 본질 (범용 도구)

**URL 목록을 주기적으로 fetch 해서 응답시간/상태 모니터링하고, 임계값 넘으면 알람 쏘는 맥북앱.**

도메인 종속 로직 없음. 어떤 사이트의 API든 URL만 등록하면 동작.

### 핵심 기능

- URL 목록 등록/수정/삭제 (또는 JSON import)
- 주기 fetch (간격 설정 가능)
- 응답시간/상태코드 SQLite 기록
- 차트 UI (각 URL별 최근 N시간 라인 차트)
- 임계값 2단계: 주의 / 심각
- 알람: 슬랙 webhook + macOS 네이티브 알림
- Dock 아이콘으로 상태 글랜서블 표시

### 기술 스택 (Decided)

| 항목 | 선택 | 이유 |
|---|---|---|
| 플랫폼 | Electron 맥북앱 | UI 글랜서블, 측정+UI 자급자족, 자동시작 쉬움 |
| 프론트 | React + TypeScript | 차트 라이브러리 풍부 |
| 차트 | Recharts | D3 풀스택은 오버킬 |
| 데이터 저장 | SQLite (better-sqlite3) | 히스토리 영속. 메모리만이면 누수 위험 |
| 알람 | 슬랙 Webhook | 가장 단순 |
| 자동 실행 | macOS "로그인 시 자동 실행" | launchd 별도 안 씀 |

**메모리 부담**: 단순 fetch + 차트라 80~200MB 예상.

## 2. 기능 요구사항

### MVP (1차)
- [x] URL 목록 관리 UI (추가/수정/삭제/JSON import)
- [x] 각 URL에 라벨/메모/그룹(태그) 달기
- [x] 30초 간격 fetch (간격은 설정에서 조절)
- [x] 응답시간(ms) + 상태코드 SQLite 기록
- [x] 메인 창: 각 URL별 최근 1시간 라인 차트 + 현재 상태 카드 (🟢🟡🔴)
- [x] 임계값 2단계: 주의 / 심각
- [x] 임계값 넘으면 슬랙 webhook + macOS 네이티브 알림
- [x] 설정 화면: 간격, 임계값, 슬랙 webhook URL
- [x] Dock 아이콘 상태 배지

### 2차 (나중에)
- [x] 알람 피로 방지: 연속 N회 임계값 넘을 때만 알람 + 쿨다운
- [x] measurement retention (오래된 데이터 자동 삭제)
- [ ] 일/주간 리포트
- [ ] 그룹별 통합 차트
- [ ] 임의 HTTP 메서드/헤더/바디 지원 (1차는 GET 기본)

## 3. 운영 시나리오

1. **개발**: 프로젝트 디렉터리에서 `yarn dev`
2. **빌드**: `yarn build:mac` → `.dmg`
3. **배포**: 맥북에 빌드본 옮겨 실행 + 로그인 시 자동시작 설정
4. **모니터링 대상 입력**: UI에서 직접 추가 또는 JSON import
5. **알람 수신**: 슬랙 채널 (처음엔 별도 채널에 쌓다가 임계값 튜닝 후 본 채널)

### 임계값 튜닝 단계
1. **첫 1주**: 알람 끄고 데이터만 수집 → 정상 응답시간 분포 파악
2. **임계값 설정**: p95 응답시간 × 2 또는 3 정도로 시작
3. **점진 조정**: 오탐 많으면 올리고, 놓치면 내리고

---

## 4. 주의사항

- 알람 폭격 금지. 별도 채널 → 검증 → 본 채널 단계로
- 차트 데이터 메모리 누적 주의. 최근 N시간만 메모리에, 나머지는 SQLite (retention 자동 정리)
- 타겟 사이트 부담 주의. 30초 간격 × URL 3~5개면 분당 6~10회. 더 짧게는 가지 말기
